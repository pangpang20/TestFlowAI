## 1. 开发环境搭建

### 1.1 软件要求

| 软件 | 版本 | 下载链接 |
|------|------|---------|
| JDK | 17+ | https://adoptium.net/ |
| Node.js | 18+ | https://nodejs.org/ |
| Maven | 3.8+ | https://maven.apache.org/ |
| Docker | 最新 | https://docker.com/ |
| IDE | IntelliJ IDEA / VS Code | - |
| Git | 最新 | https://git-scm.com/ |

### 1.2 克隆项目

```bash
git clone https://github.com/pangpang20/TestFlowAI.git
cd TestFlowAI
```

### 1.3 启动基础设施

```bash
cd docker

# 启动 MySQL
docker-compose up -d mysql

# 启动 MinIO（文件存储）
docker-compose up -d minio

# 验证容器状态
docker-compose ps
```

### 1.4 初始化数据库

```bash
# 执行初始化脚本
docker exec mysql mysql < init-sql/init.sql
```

### 1.5 后端开发环境

```bash
cd backend

# 安装依赖
mvn clean install

# 配置开发环境
# 编辑 src/main/resources/application-dev.yml

# 启动服务
mvn spring-boot:run

# 或使用 JAR 方式
mvn clean package
java -jar target/backend-1.0.0-SNAPSHOT.jar
```

**配置检查：**
- 数据库连接信息
- JWT 密钥配置
- CORS 允许的源

### 1.6 前端开发环境

```bash
cd frontend

# 安装依赖
npm install

# 配置环境变量
# 编辑 .env 文件
VITE_API_BASE_URL=http://localhost:18080

# 启动开发服务器
npm run dev

# 构建生产版本
npm run build
```

### 1.7 Chrome 扩展开发

```bash
cd chrome-extension

# 加载扩展
# 1. 打开 chrome://extensions/
# 2. 开启开发者模式
# 3. 点击"加载已解压的扩展程序"
# 4. 选择 chrome-extension 目录
```

## 3. 数据库设计

### 3.1 数据库表概览

| 表名 | 说明 | 主要字段 |
|------|------|---------|
| t_user | 用户表 | user_id, username, password, email, status |
| t_role | 角色表 | role_id, role_name, role_code |
| t_permission | 权限表 | permission_id, permission_name, permission_code |
| t_user_role | 用户角色关联表 | user_id, role_id |
| t_role_permission | 角色权限关联表 | role_id, permission_id |
| t_project | 项目表 | project_id, name, description, status, progress |
| t_test_flow | 测试流表 | test_id, title, version, app_url, steps, variables |
| t_execution | 执行记录表 | execution_id, test_id, status, mode, start_time, end_time |
| t_report | 报告表 | report_id, execution_id, status, total_steps, passed_steps |
| t_scheduled_task | 定时任务表 | task_id, task_name, test_id, cron_expression, status |
| t_audit_log | 审计日志表 | log_id, user_id, action, resource, timestamp |

### 3.2 核心表结构

#### 用户表 (t_user)

```sql
CREATE TABLE t_user (
    user_id VARCHAR(64) PRIMARY KEY,
    username VARCHAR(50) NOT NULL UNIQUE,
    password VARCHAR(255) NOT NULL,
    email VARCHAR(100),
    avatar VARCHAR(255),
    status VARCHAR(20) DEFAULT 'active',
    last_login_at DATETIME,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    deleted_at DATETIME,
    deleted TINYINT DEFAULT 0,
    created_by VARCHAR(64),
    updated_by VARCHAR(64),
    deleted_by VARCHAR(64)
);
```

#### 测试流表 (t_test_flow)

```sql
CREATE TABLE t_test_flow (
    test_id VARCHAR(64) PRIMARY KEY,
    title VARCHAR(200) NOT NULL,
    version VARCHAR(50),
    app_url VARCHAR(500),
    steps TEXT,           -- JSON 格式的测试步骤
    variables TEXT,       -- JSON 格式的变量定义
    tags VARCHAR(500),    -- JSON 格式的标签数组
    expected_report TEXT, -- JSON 格式的预期报告
    project_id VARCHAR(64),
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    deleted_at DATETIME,
    deleted TINYINT DEFAULT 0,
    created_by VARCHAR(64),
    updated_by VARCHAR(64),
    deleted_by VARCHAR(64)
);
```

#### 执行记录表 (t_execution)

```sql
CREATE TABLE t_execution (
    execution_id VARCHAR(64) PRIMARY KEY,
    test_id VARCHAR(64) NOT NULL,
    mode VARCHAR(20),           -- manual/auto/scheduled
    status VARCHAR(20),         -- pending/running/passed/failed/stopped
    start_time DATETIME,
    end_time DATETIME,
    total_steps INT DEFAULT 0,
    passed_steps INT DEFAULT 0,
    failed_steps INT DEFAULT 0,
    input TEXT,                 -- JSON 格式的执行输入
    output TEXT,                -- JSON 格式的执行输出
    step_results TEXT,          -- JSON 格式的步骤结果
    screenshots TEXT,           -- JSON 格式的截图路径
    loop_context TEXT,          -- JSON 格式的循环上下文
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    deleted_at DATETIME,
    deleted TINYINT DEFAULT 0,
    created_by VARCHAR(64),
    updated_by VARCHAR(64),
    deleted_by VARCHAR(64)
);
```

@Service
public class TestEntityService {

    public TestEntity getById(String id) {

@RestController

}
