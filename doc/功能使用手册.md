## 1. 产品概述

TestFlowAI 是一个基于 AI 智能体的自动化测试流程管理平台，提供测试用例录制、编辑、执行、报告和定时任务等完整功能。

### 1.1 核心功能

| 功能模块 | 描述 |
|---------|------|
| 📁 项目管理 | 组织和管理测试项目，支持项目分类和进度跟踪 |
| 🎬 测试流录制 | Chrome 扩展录制用户操作，自动生成测试步骤 |
| ✏️ 测试流编辑 | 可视化编辑测试步骤、变量和预期结果 |
| ▶️ 测试执行 | 支持手动执行、自动执行和定时执行 |
| 📊 执行跟踪 | 实时查看测试执行进度和步骤结果 |
| 📈 测试报告 | 自动生成详细的测试报告和统计分析 |
| ⏰ 定时任务 | 配置 Cron 表达式，自动执行测试任务 |
| 👥 用户权限 | RBAC 角色权限管理，支持多用户协作 |

### 1.2 技术架构

```
┌─────────────────────────────────────────────────────────────┐
│                        前端 (Vue 3)                          │
│  Dashboard | 项目管理 | 测试流 | 执行历史 | 定时任务 | 报告   │
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────┐
│                      后端 (Spring Boot)                      │
│   Controller → Service → Mapper → MySQL/MyBatis            │
│   Security (JWT) | CORS | WebSocket | Playwright           │
└─────────────────────────────────────────────────────────────┘
                            │
         ┌──────────────────┼──────────────────┐
         ▼                  ▼                  ▼
    ┌──────────┐      ┌──────────┐      ┌──────────┐
    │  MySQL   │      │  MinIO   │      │ Playwright│
    │  数据库   │      │ 文件存储  │      │ 浏览器自动化│
    └──────────┘      └──────────┘      └──────────┘
```

## 2. 快速开始

### 2.1 环境要求

| 组件 | 版本要求 | 用途 |
|------|---------|------|
| JDK | 17+ | 后端运行 |
| Node.js | 18+ | 前端构建 |
| Maven | 3.8+ | 依赖管理 |
| Docker | 最新 | 容器化部署 |
| Chrome | 100+ | 浏览器录制 |

### 2.2 启动服务

#### 步骤 1：启动基础设施（Docker）

```bash
cd /opt/TestFlowAI/docker

# 启动 MySQL 和 MinIO
docker-compose up -d mysql minio

# 查看容器状态
docker-compose ps
```

#### 步骤 2：启动后端服务

```bash
cd /opt/TestFlowAI/backend

# 方式 1：Maven 运行
mvn spring-boot:run

# 方式 2：JAR 包运行
java -jar target/backend-1.0.0-SNAPSHOT.jar

# 查看日志
tail -f /tmp/backend.log
```

**后端配置检查：**
- 服务端口：`18080`
- 数据库连接：`jdbc:mysql://localhost:3306/testflowai`
- 日志级别：DEBUG（开发环境）

#### 步骤 3：启动前端服务

```bash
cd /opt/TestFlowAI/frontend

# 安装依赖（首次运行）
npm install

# 启动开发服务器
npm run dev

# 查看日志
tail -f /tmp/frontend.log
```

**前端配置检查：**
- 服务端口：`3001`
- API 地址：`http://172.16.1.137:18080`（在 `.env` 中配置）

#### 步骤 4：访问应用

打开浏览器访问：
- 本地访问：`http://localhost:3001`
- 局域网访问：`http://172.16.1.137:3001`

**默认管理员账户：**
- 用户名：`admin`
- 密码：`password`

## 5. 测试流管理

### 5.1 什么是测试流

测试流（Test Flow）是 TestFlowAI 的核心概念，代表一个完整的测试场景。一个测试流包含：

- **基本信息**：标题、版本号、应用 URL、标签
- **测试步骤**：按顺序执行的浏览器操作
- **变量定义**：测试过程中使用的变量
- **预期报告**：期望的测试结果

### 5.2 创建测试流

#### 方式一：手动创建

1. 进入 **测试管理** 页面
2. 点击 **新建测试流** 按钮
3. 填写基本信息：
   - **测试流名称**：必填，如"用户登录测试"
   - **版本号**：必填，如"1.0.0"
   - **应用 URL**：被测应用的地址，如"http://localhost:3000"
   - **标签**：可选，用逗号分隔，如"登录，冒烟测试"
4. 配置测试步骤（JSON 格式）：

```json
[
  {
    "type": "navigate",
    "value": "http://localhost:3000/login",
    "description": "打开登录页面"
  },
  {
    "type": "input",
    "selector": "#username",
    "value": "admin",
    "description": "输入用户名"
  },
  {
    "type": "input",
    "selector": "#password",
    "value": "admin123",
    "description": "输入密码"
  },
  {
    "type": "click",
    "selector": "#loginBtn",
    "description": "点击登录按钮"
  },
  {
    "type": "assert",
    "selector": ".welcome",
    "value": "欢迎",
    "description": "验证登录成功"
  }
]
```

5. 配置变量定义（可选）：

```json
{
  "username": "admin",
  "password": "admin123",
  "expectedWelcome": "欢迎"
}
```

6. 配置预期报告（可选）：

```json
{
  "status": "passed",
  "totalSteps": 5,
  "expectedDuration": 10000
}
```

7. 点击 **确定** 保存

#### 方式二：Chrome 扩展录制

详见 [Chrome 录制扩展](#11-chrome 录制扩展) 章节

### 5.3 测试步骤类型说明

| 类型 | 描述 | 参数说明 |
|------|------|---------|
| `navigate` | 打开 URL | `value`: 目标 URL |
| `click` | 单击元素 | `selector`: CSS 选择器 |
| `dblclick` | 双击元素 | `selector`: CSS 选择器 |
| `input` | 输入文本 | `selector`: 输入框选择器，`value`: 输入内容 |
| `select` | 选择下拉选项 | `selector`: 下拉框选择器，`value`: 选项值 |
| `check` | 勾选复选框 | `selector`: 复选框选择器 |
| `uncheck` | 取消勾选复选框 | `selector`: 复选框选择器 |
| `scroll` | 滚动页面 | `selector`: 元素选择器（可选） |
| `hover` | 鼠标悬停 | `selector`: 元素选择器 |
| `keypress` | 按键操作 | `selector`: 元素选择器，`value`: 按键值 |
| `assert` | 断言验证 | `selector`: 元素选择器，`value`: 期望内容 |

### 5.4 编辑测试流

1. 在测试流列表中找到目标测试流
2. 点击 **编辑** 按钮
3. 修改测试流信息
4. 点击 **确定** 保存

### 5.5 复制测试流

1. 点击测试流行上的 **复制** 按钮
2. 系统将创建一个相同配置的新测试流
3. 新测试流名称为"原名称_副本"

### 5.6 删除测试流

1. 点击测试流行上的 **删除** 按钮
2. 确认删除操作
3. 测试流将被软删除

### 5.7 搜索测试流

在搜索框中输入测试流名称关键词，实时筛选匹配结果。

## 6. 测试执行

### 6.1 执行方式

TestFlowAI 支持三种执行方式：

| 方式 | 描述 | 适用场景 |
|------|------|---------|
| 手动执行 | 用户在页面点击执行按钮 | 临时测试、调试 |
| 自动执行 | 创建执行记录后自动开始 | 集成测试流程 |
| 定时执行 | 按 Cron 表达式自动执行 | 每日构建、回归测试 |

### 6.2 手动执行测试流

1. 进入 **测试管理** 页面
2. 找到目标测试流
3. 点击 **执行** 按钮（如已实现）
4. 查看实时执行日志和进度

### 6.3 执行状态说明

| 状态 | 描述 |
|------|------|
| `pending` | 等待执行 |
| `running` | 正在执行 |
| `passed` | 执行通过 |
| `failed` | 执行失败 |
| `stopped` | 已停止 |

### 6.4 实时执行日志

执行过程中，系统会实时显示：
- 当前执行的步骤
- 步骤执行结果（通过/失败）
- 步骤执行时间
- 错误信息（如有）
- 执行截图（如已配置）

## 8. 测试报告

### 8.1 报告生成

测试执行完成后，系统自动生成测试报告，包含：

- **执行概览**：执行 ID、测试流、执行时间、状态等
- **步骤详情**：每个步骤的执行结果、耗时、截图
- **统计信息**：总步骤数、通过数、失败数、通过率
- **环境信息**：浏览器版本、执行机器等

### 8.2 查看报告

1. 进入 **执行历史** 页面
2. 找到已完成的执行记录
3. 点击 **查看报告** 按钮
4. 查看详细的测试报告

### 8.3 下载报告

1. 在报告详情页点击 **下载报告** 按钮
2. 下载 JSON 格式的报告文件
3. 可用于离线分析或归档

### 8.4 报告指标说明

| 指标 | 说明 |
|------|------|
| 总步骤数 | 测试流中定义的步骤总数 |
| 通过步骤数 | 执行成功的步骤数 |
| 失败步骤数 | 执行失败的步骤数 |
| 通过率 | 通过步骤数 / 总步骤数 × 100% |
| 执行耗时 | 从开始到结束的总耗时（毫秒） |
| 平均步骤耗时 | 总耗时 / 总步骤数 |

## 9. 定时任务

### 9.1 创建定时任务

1. 进入 **定时任务** 页面
2. 点击 **新建任务** 按钮
3. 填写任务信息：
   - **任务名称**：必填，如"每日冒烟测试"
   - **测试流**：选择要执行的测试流
   - **任务标题**：任务的简短描述
   - **任务描述**：详细说明（可选）
   - **Cron 表达式**：执行时间规则
   - **任务状态**：active（启用）/ inactive（禁用）

4. 点击 **确定** 保存

### 9.2 Cron 表达式格式

Cron 表达式由 5 个字段组成：

```
┌───────────── 分钟 (0 - 59)
│ ┌───────────── 小时 (0 - 23)
│ │ ┌───────────── 日 (1 - 31)
│ │ │ ┌───────────── 月 (1 - 12)
│ │ │ │ ┌───────────── 星期 (0 - 6) (周日 = 0)
│ │ │ │ │
* * * * *
```

#### 常用示例

| 表达式 | 说明 |
|--------|------|
| `0 9 * * *` | 每天早上 9 点执行 |
| `0 9 * * 1-5` | 工作日早上 9 点执行 |
| `0 */2 * * *` | 每 2 小时执行一次 |
| `0 0 12 * * ?` | 每天中午 12 点执行 |
| `0 15 10 ? * MON-FRI` | 周一至周五上午 10:15 执行 |
| `0 0/5 14 * * ?` | 每天 14:00-14:59，每 5 分钟执行 |

### 9.3 编辑定时任务

1. 在任务列表中找到目标任务
2. 点击 **编辑** 按钮
3. 修改任务配置
4. 点击 **确定** 保存

### 9.4 启用/禁用任务

1. 点击任务状态切换按钮
2. 任务状态在 active 和 inactive 之间切换
3. 禁用的任务不会自动执行

### 9.5 手动触发任务

1. 点击任务行上的 **立即执行** 按钮
2. 任务将立即执行一次（不受 Cron 表达式限制）

### 9.6 删除定时任务

1. 点击任务行上的 **删除** 按钮
2. 确认删除操作
3. 任务将被删除

## 11. Chrome 录制扩展

### 11.1 安装扩展

1. 打开 Chrome 浏览器
2. 访问扩展管理页面：`chrome://extensions/`
3. 打开右上角 **开发者模式** 开关
4. 点击 **加载已解压的扩展程序**
5. 选择扩展目录：`/opt/TestFlowAI/chrome-extension`
6. 扩展安装完成

### 11.2 使用扩展录制

1. 点击浏览器工具栏中的 TestFlowAI 图标
2. 点击 **开始录制** 按钮
3. 在目标网页上执行操作（点击、输入等）
4. 点击 **停止录制** 按钮
5. 填写测试流信息：
   - 测试流名称
   - 版本号
   - 标签
6. 点击 **保存** 按钮
7. 测试流将自动保存到 TestFlowAI 系统

### 11.3 录制操作说明

| 操作 | 录制的步骤类型 |
|------|--------------|
| 点击元素 | `click` |
| 双击元素 | `dblclick` |
| 输入文本 | `input` |
| 选择下拉 | `select` |
| 勾选复选框 | `check` |
| 取消勾选 | `uncheck` |
| 打开新 URL | `navigate` |

## 12. 常见问题

### 12.1 登录问题

**问题：点击登录提示"没有权限访问该资源"**

解决方案：
1. 确认用户名和密码正确
2. 检查后端服务是否正常运行
3. 检查浏览器控制台是否有 CORS 错误
4. 确认前端 `.env` 中的 API 地址正确

**问题：登录后自动退出**

解决方案：
1. Token 有效期为 24 小时，过期需重新登录
2. 清除浏览器缓存后重试
3. 检查系统时间是否正确

### 12.2 跨域问题

**问题：前端提示"网络错误，请检查网络连接"**

解决方案：
1. 确认后端服务端口 18080 正常监听
2. 检查前端 `.env` 中的 `VITE_API_BASE_URL` 配置
3. 确认后端 CORS 配置包含当前前端地址
4. 重启前后端服务

### 12.3 执行问题

**问题：测试执行失败，提示"浏览器启动失败"**

解决方案：
1. 确认服务器已安装 Chrome/Chromium 浏览器
2. 检查 Playwright 依赖是否完整
3. 查看后端日志 `/tmp/backend.log` 获取详细错误
4. 尝试在服务器上手动运行 Chromium

**问题：步骤执行超时**

解决方案：
1. 检查被测应用是否可访问
2. 增加步骤超时时间配置
3. 检查元素选择器是否正确
4. 查看网络请求是否缓慢

### 12.4 数据库问题

**问题：无法连接数据库**

解决方案：
1. 确认 MySQL 容器正常运行：`docker-compose ps`
2. 检查数据库连接配置
3. 确认数据库用户权限
4. 查看数据库日志

### 12.5 文件上传问题

**问题：头像上传失败**

解决方案：
1. 确认 MinIO 服务正常运行
2. 检查文件上传大小限制（默认 5MB）
3. 确认支持的图片格式（JPG/PNG）
4. 查看后端日志获取详细错误

### 12.6 技术支持

如遇到其他问题，请：
1. 查看后端日志：`/tmp/backend.log`
2. 查看前端日志：`/tmp/frontend.log`
3. 查看浏览器控制台错误（F12）
4. 联系系统管理员
